## Design Context

### Users
`diffx` is primarily for developers and vibe coders reviewing local git diffs. The main workflow is a human inspecting, understanding, and iterating on AI-generated or AI-assisted code quickly without losing confidence in what changed. The interface should support fast scanning, clear comparison, lightweight annotation, and low-friction git actions in one focused workspace.

### Brand Personality
The product should feel precise, fast, and legible. Its tone should be technical without becoming cold, and calm without becoming dull. The emotional goal is to help users feel oriented and in control while they review code written by AI and decide what to keep, change, or stage next.

### Aesthetic Direction
Keep the UI single-theme for now, aligned with the existing dark-first implementation. Favor a restrained technical aesthetic: dark surfaces, crisp contrast, subtle blue accents, clear hierarchy, and minimal visual noise. Motion is welcome when it improves comprehension, orientation, or feedback, but should never add latency, visual clutter, or drag on interaction speed. Avoid decorative flourish that makes the product feel playful, marketing-heavy, or vague.

### Design Principles
1. Optimize for instant comprehension.
   Every major screen should make it obvious what changed, what is selected, what action is available, and what state the repo is in.

2. Preserve technical precision.
   Visual styling should reinforce trust in the diff and git state, not compete with it. Favor exact labels, stable layout, and clear state transitions.

3. Keep interaction fast and low-friction.
   Prioritize responsive controls, efficient scanning, and minimal ceremony around common review tasks like switching files, comparing refs, annotating, staging, and committing.

4. Make AI-written code easier for humans to review.
   Design should reduce ambiguity and cognitive load so humans can quickly understand machine-generated changes and iterate with confidence.

5. Use motion only in service of clarity.
   Animation should orient, confirm, or soften transitions. If an effect does not improve usability or understanding, it should not ship.
